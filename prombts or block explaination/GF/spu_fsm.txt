params:
None. 

local params:
FSM state encodings (IDLE, PYRAMID_PROCESSING, LAYER_SWITCHING).

inputs:
clk, rst_n
frame_nums (4-bit bus containing indices for the current and previous frames)
start_spu (trigger signal from TAU)
layer_done (notification indicating read/write processing for the current layer is complete)

outputs:
idle_spu (status flag)
mem_read_cmd (pulse to trigger the memory read module)
Configuration outputs (replacing calculated addresses):

frame_i_idx (2 bits)

frame_i_minus_1_idx (2 bits)

current_layer (3 bits)

has_flow (1 bit flag determining if an upper optical flow layer exists to be fetched)

procedure:
It uses a counter register to track current_layer (from 4 down to 0) and combinational logic to determine has_flow (true if current_layer < 4). Its states are as follows:

idle:
When it receives start_spu, it pulls down idle_spu, extracts and stores the frame indices from frame_nums, sets the layer counter to the smallest layer (4), and goes to the pyramid processing state.

pyramid processing:
Issues the mem_read_cmd pulse. Because the FSM continuously outputs the configuration (frame_i_idx, frame_i_minus_1_idx, current_layer, has_flow), the Memory Read Module directly receives the parameters it needs to calculate addresses itself. The FSM then immediately goes to the layer switching state.

layer switching:
Monitors the layer_done completion signal.
Once a layer is complete, the FSM checks its layer counter.
If Layer > 0, it decrements the layer counter and jumps back to the pyramid processing state.
If Layer == 0, it means the full GF optical flow vector field has been calculated for the frame pair. It pulls idle_spu high and returns back to the idle state.
